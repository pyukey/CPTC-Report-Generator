As part of the normal operation of Kerberos, an Authentication Service AS response message is transmitted between a kerberos server and a user's client when a user requests a TGT Ticket Granting Ticket from the Domain Controller.<br><br>When the TGT is requested, a session key is also received from the KDC Key Distribution Center on the Domain Controller, which is encrypted with a user's password. Pre-authentication is something that requires a TGT requestor to prove their identity before the KDC will issue a ticket.<br><br>When no pre-authentication is enabled, which does sometimes have legitimate use cases for backwards-compatibility with older versions of kerberos, any user can request a user's hash without credentials.<br><br>
