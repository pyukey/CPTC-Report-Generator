When an SUID bit is set on an executable file in Linux, the file can be run with permissions of the file owner. If the file owner is root, a user can run the file with root privileges, allowing for privilege escalation. The <code>systemctl</code> binary had the SUID bit set, allowing users with lower privileges to execute it and escalate their privileges to the root user.<br><br>An attacker could use <code>systemctl</code> to execute commands as the root user, potentially compromising the full system.
