The insecure CORS policy was confirmed by sending a request to the [API Endpoint] endpoint while setting a custom Origin header e.g., <code>Origin: https://evil.com</code>. The server's response included the header&nbsp;<code>Access-Control-Allow-Origin: https://evil.com</code>, proving that the application will trust any domain. Since no anti-CSRF tokens are required for state-changing operations, this allows for a cross-domain attack.
