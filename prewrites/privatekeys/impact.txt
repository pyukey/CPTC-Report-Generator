If the SSH private key is used with remote servers, the attacker could use it to log into those systems and fully compromise them.<br><br>Skilled attackers would also be able to use the exposed SSH key to laterally move within the network, accessing additional company systems and thus information.<br><br>Furthermore, attackers with access to the private key can now use to intercept data sent using SSH, which could result in the company losing sensitive data and potentially violating compliance regulations.
