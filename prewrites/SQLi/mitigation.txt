Sanitize all forms or fields that accept user input. This technique involves ensuring that the user input does not contain special characters that are part of common injections. Most modern databases provide functions for this purpose.<br><br>Restrict the possible types of input that a user can provide. This can mean only allowing lowercase letters, only numbers, or a dedicated dropdown menu of acceptable inputs. If text input is necessary, validate the input through comprehensive regex rules to reject abnormal inputs.<br><br>Restrict the permissions of the database user that handles queries coming from the web server. This user should not be allowed access to any sensitive tables or databases, delegating this access only to an approved internal administrator.<br><br>Implementing a Web Application Firewall WAF can help detect malicious input and reject any HTTP requests containing them. This solution can protect both the website and the database simultaneously.
