The /etc/shadow file stores password hashes for all user accounts, including the root account. If this file is writable by non-root users, it enables a local user to directly modify account credentials, allowing for that user to escalate their privileges by changing the root password to a known value.<br><br>With this changed root password, the local user could log in as root, allowing them to execute commands with full system privileges, install backdoors to maintain access over the system, and tamper with critical system files.<br><br>Attackers with root privileges could modify or steal sensitive company data, leading to compliance violations.<br><br><div>Furthermore, changes to the /etc/shadow file could be used to launch a DoS attack on the company by changing password for legitimate users, locking those users out of their accounts.</div><div><br></div><div><b>Compliance Violations:</b></div><div><ul><li>TSA 3157 7.01</li><li>TSA 3157 7.04A</li></ul></div>
